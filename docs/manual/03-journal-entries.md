# บันทึกบัญชี (Journal Entries)
# Journal Entries Manual

**Section 3 of the Thai Accounting ERP User Manual**

---

## 1. ภาพรวม

บันทึกบัญชีคือการบันทึกรายการค้าทางการเงินตามหลัก **บัญชีคู่ (Double-Entry)** ทุกรายการต้องมีเดบิต = เครดิต

---

## 2. สถานะบันทึกบัญชี (Entry Status)

| สถานะ | Thai | ความหมาย |
|-------|------|---------|
| **DRAFT** | ฉบับร่าง | บันทึกแล้วแต่ยังไม่ผ่านรายการ |
| **POSTED** | ผ่านรายการแล้ว | บันทึกเข้าระบบบัญชีเรียบร้อย |
| **VOID** | ยกเลิก | รายการถูกยกเลิก (ไม่ลบ แต่ตีกลับ) |

---

## 3. การสร้างบันทึกบัญชีใหม่

**ทางเข้า:** เมนูด้านข้าง → **บัญชี** → **บันทึกบัญชี** → **+ บันทึกใหม่**

### ขั้นตอน
1. **เลือกวันที่** — วันที่รายการเกิด (ต้องอยู่ในงวดบัญชีที่เปิดอยู่)
2. **กรอกรายละเอียด** — คำอธิบายรายการ
3. **เพิ่มรายการบัญชี** — แต่ละรายการต้องมี:
   - **เลือกบัญชี** — จากผังบัญชี
   - **รายละเอียด** — คำอธิบายเพิ่มเติม (ถ้ามี)
   - **เดบิต** — จำนวนเงินด้านเดบิต (บาท)
   - **เครดิต** — จำนวนเงินด้านเครดิต (บาท)

4. **ตรวจสอบยอด** — ระบบแสดง "ยอดเดบิต = ยอดเครดิต" ทุกครั้ง
5. **บันทึก** — ระบบจะสร้างเลขที่บันทึกอัตโนมัติ (เช่น JE-2026-0001)

### หลักการบันทึก
- **เดบิต** อยู่ทางซ้าย, **เครดิต** อยู่ทางขวา
- ถ้าเลือกเดบิต → เครดิต = 0 (และในทางกลับกัน)
- รายการต้องมีอย่างน้อย 2 บัญชี

### ตัวอย่าง: จ่ายค่าเช่าสำนักงาน ฿15,000
```
เดบิต  ค่าเช่าสำนักงาน    ฿15,000
  เครดิต  เงินสด            ฿15,000
```

---

## 4. การผ่านรายการ (Post)

เมื่อบันทึกบัญชีแล้ว ต้อง **ผ่านรายการ** เพื่อให้เข้าสู่ระบบบัญชีจริง

### ขั้นตอน
1. เปิดบันทึกบัญชีที่มีสถานะ **DRAFT**
2. กดปุ่ม **ผ่านรายการ (Post)**
3. ระบบจะตรวจสอบ:
   - งวดบัญชีต้องเปิดอยู่
   - ยอดเดบิต = ยอดเครดิต
4. กด **ยืนยัน** → สถานะเปลี่ยนเป็น **POSTED**

> ⚠️ หลังผ่านรายการแล้ว **ไม่สามารถแก้ไขได้** — ต้องยกเลิกแล้วสร้างใหม่

---

## 5. การยกเลิกบันทึกบัญชี (Void)

ถ้าบันทึกผิด สามารถ **ยกเลิก** ได้โดย:

### ขั้นตอน
1. เปิดบันทึกบัญชีที่มีสถานะ **POSTED**
2. กดปุ่ม **ยกเลิก (Void)**
3. ระบบจะสร้าง **บันทึกกลับ (Reversal Entry)** อัตโนมัติ:
   - สลับเดบิต ↔ เครดิต
   - เลขที่ใหม่ต่อท้ายด้วย "-R"
   - เลขายนั้นจะเปลี่ยนเป็น **VOID**

### ตัวอย่าง: ยกเลิกรายการจ่ายค่าเช่า
```
บันทึกต้นฉบับ (VOID):
เดบิต  ค่าเช่าสำนักงาน    ฿15,000
  เครดิต  เงินสด            ฿15,000

บันทึกกลับ (POSTED):
เดบิต  เงินสด            ฿15,000
  เครดิต  ค่าเช่าสำนักงาน    ฿15,000
```

---

## 6. การอนุมัติตามหลัก Separation of Duties

ตามระบบ RBAC ที่ตั้งไว้:

| บทบาท | สร้างบันทึก | ผ่านรายการ |
|-------|-----------|-----------|
| ACCOUNTANT | ✅ | ❌ |
| SENIOR_ACCOUNTANT | ✅ | ✅ |
| ADMIN/OWNER | ✅ | ✅ |

**หลักการ:** ACCOUNTANT สร้างรายการ → SENIOR_ACCOUNTANT หรือ ADMIN ผ่านรายการ

---

## 7. รายการอัตโนมัติ (Auto-Journal)

ระบบจะสร้างบันทึกบัญชีอัตโนมัติเมื่อ:

| เอกสาร | การสร้างบัญชีอัตโนมัติ |
|--------|---------------------|
| ใบแจ้งหนี้ (Invoice) | ผ่านรายการ → สร้าง Journal Entry ทันที |
| ใบเสร็จ (Receipt) | ผ่านรายการ → สร้าง Journal Entry ทันที |
| ชำระเงิน (Payment) | ผ่านรายการ → สร้าง Journal Entry ทันที |
| ค่าเสื่อมราคา (Depreciation) | รันเดือนละครั้ง อัตโนมัติ |
| เงินเดือน (Payroll) | ผ่านรายการ → สร้าง Journal Entry ทันที |

---

## 8. การค้นหาและกรอง

### ค้นหาด่วน
- พิมพ์ **เลขที่บันทึก** หรือ **คำอธิบาย** ในช่องค้นหา

### กรองตามสถานะ
- ทั้งหมด / DRAFT / POSTED / VOID

### กรองตามวันที่
- ระบุวันที่เริ่มต้น → สิ้นสุด
- ใช้ดูรายการในงวดบัญชีเฉพาะ
