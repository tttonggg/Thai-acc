# งวดบัญชี (Accounting Periods)
# Accounting Periods Manual

**Section 4 of the Thai Accounting ERP User Manual**

---

## 1. ภาพรวม

งวดบัญชีคือช่วงเวลาที่ใช้ในการบันทึกรายการบัญชี ตามกฎหมายไทย ธุรกิจต้องจัดทำงบการเงินเป็น **รายเดือน หรือ รายวันที่สิ้นสุดของรอบระยะเวลาบัญชี**

### ประเภทงวดบัญชี
- **รายเดือน** — นิยมใช้ในธุรกิจ SME
- **รายไตรมาส** — การจัดทำ VAT รายไตรมาส (PP30/PP36)
- **รายปี** — ปิดบัญชีสิ้นปีบัญชี (มกราคม-ธันวาคม หรือเดือนอื่นตามรอบบัญชี)

---

## 2. สถานะงวดบัญชี (Period Status)

| สถานะ | Thai | ความหมาย | การบันทึกบัญชี |
|-------|------|---------|--------------|
| **OPEN** | เปิดอยู่ | สามารถบันทึกรายการได้ | ✅ บันทึกได้ปกติ |
| **ADJUSTING** | ปิดปรับปรุง | กำลังปรับปรุงรายการ | ⚠️ จำกัดการแก้ไข |
| **CLOSED** | ปิดแล้ว | ไม่อนุญาตให้แก้ไข | ❌ ห้ามบันทึกเพิ่ม |

### ความหมายของแต่ละสถานะ

**OPEN (เปิดอยู่)**
- บันทึกรายการได้ตามปกติ
- ผ่านรายการได้
- แก้ไขรายการได้ (ยังไม่ปิดงวด)

**ADJUSTING (ปิดปรับปรุง)**
- งวดบัญชีสิ้นเดือนแล้ว แต่ยังมีการปรับปรุง
- ใช้สำหรับการปิดสิ้นเดือน (Month-End Adjustment)
- รายการปรับปรุงยังได้

**CLOSED (ปิดแล้ว)**
- ไม่สามารถเพิ่มหรือแก้ไขรายการได้
- ต้อง **เปิดใหม่** ก่อนถ้าต้องการแก้ไข
- ต้องมีสิทธิ์ **ADMIN/OWNER** เท่านั้น

---

## 3. การจัดการงวดบัญชี

**ทางเข้า:** เมนูด้านข้าง → **บัญชี** → **งวดบัญชี**

### 3.1 สร้างงวดบัญชีใหม่
1. กดปุ่ม **+ สร้างงวดใหม่**
2. กรอกข้อมูล:
   - **ชื่องวด** — เช่น "มกราคม 2569" หรือ "ไตรมาสที่ 1/2569"
   - **วันที่เริ่มต้น** — วันแรกของงวด
   - **วันที่สิ้นสุด** — วันสุดท้ายของงวด
   - **สถานะ** — เริ่มต้นเป็น OPEN
3. กด **บันทึก**

### 3.2 เปิด/ปิดงวดบัญชี
> ⚠️ ต้องมีสิทธิ์ **ADMIN/OWNER** เท่านั้น

**ปิดงวดบัญชี:**
1. เลือกงวดที่ต้องการปิด
2. กดปุ่ม **ปิดงวด (Close Period)**
3. ระบบจะตรวจสอบ:
   - งวดก่อนหน้าต้องปิดแล้ว
   - ไม่มีรายการค้างค้าง (optional)
4. กด **ยืนยัน** → สถานะเปลี่ยนเป็น **CLOSED**

**เปิดงวดใหม่ (Reopen):**
> ⚠️ การเปิดงวดที่ปิดแล้วต้องมีเหตุผลอันสมควร และต้องล็อกอินด้วยสิทธิ์ ADMIN/OWNER

---

## 4. ขั้นตอนปิดสิ้นเดือน (Month-End Closing)

### Checklist ปิดงวดสิ้นเดือน

**สัปดาห์ก่อนสิ้นเดือน:**
- [ ] ตรวจสอบรายการที่ยังไม่ได้ผ่านรายการ (DRAFT)
- [ ] ตรวจสอบใบแจ้งหนี้/ใบเสร็จที่ยังไม่ออก
- [ ] ตรวจสอบการรับชำระเงินที่ยังไม่ได้บันทึก

**วันสุดท้ายของเดือน:**
- [ ] บันทึกรายการปรับปรุงปลายงวด (Adjusting Entries)
- [ ] คำนวณค่าเสื่อมราคา
- [ ] ตรวจสอบความถูกต้องของยอดธนาคาร
- [ ] ตรวจสอบยอดลูกหนี้/เจ้าหนี้

**หลังสิ้นเดือน (ภายใน 15 วัน):**
- [ ] ผ่านรายการทั้งหมด
- [ ] ปิดงวดบัญชี

## 5. การปรับปรุงปลายงวด (Adjusting Entries)

รายการปรับปรุงที่พบบ่อย:

### 5.1 ค่าเสื่อมราคาสินทรัพย์
```
เดบิต  ค่าเสื่อมราคา (ค่าใช้จ่าย)    ฿5,000
  เครดิต  ค่าเสื่อมสะสม              ฿5,000
```

### 5.2 ค่าใช้จ่ายค้างจ่าย (Accrued Expenses)
```
เดบิต  ค่าใช้จ่ายในการดำเนินงาน      ฿3,000
  เครดิต  ค่าใช้จ่ายต้องจ่าย           ฿3,000
```

### 5.3 รายได้ค้างรับ (Accrued Revenue)
```
เดบิต  ลูกหนี้ค่าบริการ              ฿10,000
  เครดิต  รายได้ค้างรับ               ฿10,000
```

### 5.4 สำรองลูกหนี้สงสัยจะสูญ
```
เดบิต  ค่าเสื่อมสงสัยจะสูญ           ฿2,000
  เครดิต  สำรองลูกหนี้สงสัยจะสูญ       ฿2,000
```

## 6. การตรวจสอบก่อนปิดงวด

### Trial Balance
- ยอดรวมเดบิต = ยอดรวมเครดิต
- ดูได้ที่เมนู **รายงาน** → **งบทดลอง**

### รายการที่ต้องตรวจสอบ
- บัญชีเงินสด/ธนาคาร: ยอดตรงกับใบแจ้งยอดธนาคาร
- ลูกหนี้: ไม่เกินกำหนดชำระ
- เจ้าหนี้: ตรวจสอบกับใบสั่งซื้อ
- สินค้าคงเหลือ: ตรวจสอบ Stock Take

---

## 7. งวดบัญชีและภาษี

### ความสัมพันธ์กับ VAT
- งวดบัญชีใช้กำหนดว่ารายการนั้นอยู่ในงวดใด
- VAT รายเดือน (PP30) ต้องยื่นภายในวันที่ 15 ของเดือนถัดไป
- VAT รายไตรมาส (PP36) ยื่นพร้อมแบบแสดงรายการ

### ความสัมพันธ์กับ WHT
- ภาษีหัก ณ ที่จ่าย ต้องยื่นภายใน 7 วัน หลังจ่าย
- แบบ ภงด.3/1 และ ภงด.53/1 อยู่ในงวดที่จ่าย
