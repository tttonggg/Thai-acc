# การจัดการผู้ใช้และบทบาท (User & Role Management)
# User and Role Management Manual

**Section 1 of the Thai Accounting ERP User Manual**

---

## 1. การเข้าสู่ระบบ (Logging In)

### ขั้นตอนการ Login
1. เปิดเว็บไซต์ → หน้าจอ **เข้าสู่ระบบ** ปรากฏขึ้น
2. กรอก **อีเมล** และ **รหัสผ่าน** ที่ได้รับจากผู้ดูแลระบบ
3. กดปุ่ม **เข้าสู่ระบบ**
4. ระบบจะนำไปยัง **หน้าแดชบอร์ด** หรือ **หน้าหลักของโปรแกรม**

### ข้อควรระวัง
- รหัสผ่านต้องมีตัวอักษรตัวใหญ่ ตัวเล็ก ตัวเลข และอักขระพิเศษอย่างน้อย 8 ตัว
- หลัง login ผิด 5 ครั้ง ระบบจะล็อก 15 นาที
- หากลืมรหัสผ่าน → ติดต่อ **ผู้ดูแลระบบ (OWNER/ADMIN)** เพื่อรีเซ็ต

## 2. บทบาทและสิทธิ์ในระบบ (Roles & Permissions)

ระบบ RBAC มีบทบาทสำเร็จรูป **6 บทบาท** ดังนี้:

| บทบาท | Thai | สิทธิ์โดยสรุป |
|-------|------|-------------|
| **OWNER** | เจ้าของกิจการ | เข้าถึงทุกอย่าง + จัดการบทบาทได้ |
| **ADMIN** | ผู้ดูแลระบบ | จัดการระบบ + ผู้ใช้ + สิทธิ์ทั้งหมด |
| **SENIOR_ACCOUNTANT** | หัวหน้าบัญชี | สร้าง + อนุมัติ + ยกเลิกเอกสาร |
| **ACCOUNTANT** | นักบัญชี | สร้างเอกสารได้อย่างเดียว (ต้องมีคนอนุมัติ) |
| **USER** | พนักงานทั่วไป | ดูราคา/สินค้า/ใบเสนอราคาเท่านั้น |
| **VIEWER** | ผู้สังเกตการณ์ | ดูรายงานเท่านั้น |

### สิทธิ์แยกตามบทบาท (Permission Matrix)

#### OWNER / ADMIN (ผู้ดูแลระบบ)
- ✅ ทุกสิทธิ์ในระบบ
- ✅ จัดการบทบาทและสิทธิ์ผู้ใช้
- ✅ จัดการผู้ใช้ทั้งหมด
- ✅ ตั้งค่าระบบ, สกุลเงิน, ผังบัญชี
- ✅ ดู Audit Log

#### SENIOR_ACCOUNTANT (หัวหน้าบัญชี)
- ✅ สร้าง + อนุมัติ + ยกเลิก ใบแจ้งหนี้/ใบเสร็จ/ใบกำกับ
- ✅ สร้าง + อนุมัติ + ยกเลิก บันทึกบัญชี
- ✅ ดูรายงานการเงินทุกประเภท
- ✅ ดูข้อมูลพนักงาน, ลูกค้า, ผู้จัดจำหน่าย
- ❌ ไม่สามารถจัดการบทบาทหรือผู้ใช้
- ❌ ไม่สามารถตั้งค่าระบบ

#### ACCOUNTANT (นักบัญชี)
- ✅ สร้างใบแจ้งหนี้/ใบเสร็จ/บันทึกบัญชี/ใบสั่งซื้อ
- ✅ ดูรายงานการเงิน
- ❌ ไม่สามารถอนุมัติหรือยกเลิกเอกสารที่ตัวเองสร้าง
- ❌ ไม่สามารถจัดการบทบาท/สิทธิ์

#### USER (พนักงานทั่วไป)
- ✅ ดูรายการลูกค้า/สินค้า/ราคา
- ✅ สร้างใบเสนอราคา
- ✅ ดูรายงาน
- ❌ ไม่สามารถสร้างเอกสารทางบัญชี
- ❌ ไม่สามารถดูต้นทุน/กำไร

#### VIEWER (ผู้สังเกตการณ์)
- ✅ ดูรายงานการเงิน (สรุป)
- ❌ ไม่สามารถดูรายละเอียดเอกสาร
- ❌ ไม่สามารถสร้าง/แก้ไขอะไร

## 3. หน้าจัดการบทบาท (Role Management Page)

**ทางเข้า:** เมนูด้านข้าง → **ผู้ดูแลระบบ** → **จัดการบทบาท**

> ⚠️ ต้องมีสิทธิ์ `admin:manage` (OWNER หรือ ADMIN เท่านั้น)

### 3.1 หน้าจอหลัก
หน้าจอแบ่งเป็น 2 ส่วน:
- **บทบาทสำเร็จรูป (System Roles)** — สร้างโดยระบบ ไม่สามารถลบได้
- **บทบาทที่กำหนดเอง (Custom Roles)** — สร้างใหม่ได้โดยผู้ดูแล

### 3.2 ดูรายละเอียดบทบาท
- คลิกที่ **ชื่อบทบาท** → ขยายแสดงสิทธิ์ที่บทบาทนี้มี
- แสดงจำนวน **ผู้ใช้ที่ใช้บทบาทนี้**

### 3.3 เพิ่ม/ลบสิทธิ์ในบทบาท (เฉพาะ Custom Roles)
1. คลิกเลือก Custom Role ที่ต้องการแก้ไข
2. หา **สิทธิ์ที่ต้องการ** ในรายการด้านล่าง
3. กด **Toggle** เพื่อเปิด/ปิดสิทธิ์
4. ระบบจะอัปเดตทันที

### 3.4 สร้างบทบาทใหม่
1. เลื่อนไปด้านล่าง → กด **+ สร้างบทบาทใหม่**
2. กรอกชื่อบทบาท (ภาษาไทย/อังกฤษ)
3. เลือกสิทธิ์ที่ต้องการ
4. กด **บันทึก**

## 4. การปรับแต่งบทบาทตามวัฒนธรรมองค์กร
### Customizing Roles for Your Company

องค์กรสามารถ **สร้างบทบาทใหม่** หรือ **ปรับสิทธิ์ Custom Roles** ได้ตามโครงสร้างของตัวเอง

**ตัวอย่าง: บริษัทมีแผนกขาย**
```
พนักงานขาย (SALES)
- ดูรายการสินค้า + ราคา
- สร้างใบเสนอราคา
- ดูใบเสนอราคาของตัวเอง
- ไม่สามารถดูต้นทุน
```

**ตัวอย่าง: บริษัทมีแผนกคลัง**
```
เจ้าหน้าที่คลัง (WAREHOUSE)
- ดูรายการสินค้า
- บันทึกรับ-จ่ายสินค้า
- ดู Stock Take
- ไม่สามารถสร้างเอกสารทางบัญชี
```

## 5. หลักการความปลอดภัย (Security Best Practices)

### Separation of Duties (การแยกหน้าที่)
| หน้าที่ | สร้างเอกสาร | อนุมัติเอกสาร |
|--------|------------|--------------|
| ACCOUNTANT | ✅ | ❌ |
| SENIOR_ACCOUNTANT | ✅ | ✅ |
| ADMIN/OWNER | ✅ | ✅ |

**หลักการ:** คนที่สร้างเอกสาร **ไม่ควรเป็นคนเดียวกับ** ที่อนุมัติเอกสารนั้น

### ข้อแนะนำ
1. **ผู้ดูแลระบบ** ควรเป็นเจ้าของกิจการหรือผู้จัดการที่เชื่อถือได้
2. **ไม่ควร** ให้ USER ทั่วไปเป็น ADMIN
3. **ตรวจสอบ Audit Log** เป็นประจำ (เมนู: Audit Log)
4. **เปลี่ยนรหัสผ่าน** ทุก 90 วัน
5. **ไม่ควร** สร้างบทบาทใหม่โดยให้สิทธิ์มากเกินจำเป็น

---

## 6. การติดต่อขอสิทธิ์เพิ่ม
หากผู้ใช้ต้องการสิทธิ์เพิ่ม → ติดต่อ **ผู้ดูแลระบบ (OWNER/ADMIN)** โดยตรง โดยระบุ:
- ชื่อ-นามสกุล
- อีเมลที่ใช้งาน
- สิทธิ์ที่ต้องการ
- เหตุผลที่ต้องการสิทธิ์นั้น
