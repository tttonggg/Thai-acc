# คู่มือการนำเข้าข้อมูล (Data Import Guide)

## ภาพรวม

ระบบนำเข้าข้อมูล (Data Import System)
ช่วยให้ผู้ดูแลระบบสามารถนำเข้าข้อมูลจำนวนมากได้จากไฟล์ CSV หรือ JSON
โดยรองรับข้อมูลประเภทต่อไปนี้:

- ลูกค้า (Customers)
- ผู้ขาย (Vendors)
- สินค้า (Products)
- ผังบัญชี (Chart of Accounts)

## คุณสมบัติหลัก

### 1. การตรวจสอบข้อมูล (Data Validation)

- ตรวจสอบรูปแบบข้อมูลอัตโนมัติ
- แสดงผลการตรวจสอบล่วงหน้า (Preview)
- ระบุรายการที่ผิดพลาดพร้อมข้อความแสดงข้อผิดพลาด

### 2. ตัวเลือกการนำเข้า

- **โหมดตรวจสอบ (Dry Run)**: ตรวจสอบข้อมูลโดยไม่นำเข้าจริง
- **ข้ามรายการซ้ำ (Skip Duplicates)**: ข้ามรหัสที่มีอยู่แล้ว
- **อัปเดตข้อมูลเดิม (Update Existing)**: อัปเดตข้อมูลที่มีอยู่แล้ว

### 3. การติดตามประวัติ

- บันทึกประวัติการนำเข้าทั้งหมด
- แสดงสถิติ: ทั้งหมด, สร้างใหม่, อัปเดต, ผิดพลาด
- ดูรายละเอียดข้อผิดพลาด

## รูปแบบไฟล์

### ไฟล์ CSV

ไฟล์ CSV ต้องมี:

- **บรรทัดแรก**: ส่วนหัว (Header) - ชื่อฟิลด์
- **บรรทัดถัดไป**: ข้อมูลแต่ละรายการ
- **ตัวคั่น**: เครื่องหมายจุลภาค (`,`)
- **การเขียนในเครื่องหมายคำพูด**: ใช้เครื่องหมายคำพูดคู่ (`"`)

ตัวอย่าง:

```csv
code,name,taxId,address,phone,email
C001,บริษัท ก. จำกัด,1234567890123,"123 ถ.สุขุมวิท",02-123-4567,sales@companyg.com
```

### ไฟล์ JSON

ไฟล์ JSON ต้องเป็นอาร์เรย์ของออบเจ็กต์:

```json
[
  {
    "code": "C001",
    "name": "บริษัท ก. จำกัด",
    "taxId": "1234567890123",
    "address": "123 ถ.สุขุมวิท",
    "phone": "02-123-4567",
    "email": "sales@companyg.com"
  }
]
```

## ขั้นตอนการนำเข้า

### 1. เตรียมไฟล์ข้อมูล

1. ดาวน์โหลดไฟล์ตัวอย่าง (Template) จากระบบ
2. กรอกข้อมูลลงในไฟล์ตามรูปแบบที่กำหนด
3. บันทึกไฟล์เป็น .csv หรือ .json

### 2. อัปโหลดไฟล์

1. เลือกประเภทข้อมูลที่ต้องการนำเข้า
2. คลิกหรือลากไฟล์มาวางในพื้นที่อัปโหลด
3. รอสักครู่ให้ระบบตรวจสอบไฟล์

### 3. ตั้งค่าตัวเลือก

- **โหมดตรวจสอบ**: เลือกเพื่อดูตัวอย่างผลลัพธ์ก่อนนำเข้าจริง
- **ข้ามรายการซ้ำ**: เลือกหากต้องการข้ามรหัสที่มีอยู่แล้ว
- **อัปเดตข้อมูลเดิม**: เลือกหากต้องการอัปเดตข้อมูลที่มีอยู่

### 4. ตรวจสอบข้อมูล

1. คลิกปุ่ม "ตรวจสอบข้อมูล"
2. ดูตัวอย่างผลลัพธ์:
   - สีเขียว: จะสร้างใหม่
   - สีเหลือง: จะอัปเดต
   - สีแดง: ผิดพลาด
3. แก้ไขข้อผิดพลาดในไฟล์ต้นทาง (ถ้ามี)

### 5. นำเข้าข้อมูล

1. ถอนเลือก "โหมดตรวจสอบ"
2. คลิกปุ่ม "นำเข้าข้อมูล"
3. ยืนยันการนำเข้า
4. รอสักครู่ให้ระบบประมวลผล
5. ดูผลลัพธ์การนำเข้า

### 6. ตรวจสอบผลลัพธ์

- ดูสรุปผลการนำเข้า
- ตรวจสอบจำนวนรายการที่สร้าง/อัปเดต/ผิดพลาด
- ดูรายละเอียดข้อผิดพลาด (ถ้ามี)
- ตรวจสอบประวัติการนำเข้าในแท็บ "ประวัติ"

## ข้อควรระวัง

### ก่อนนำเข้า

- สำรองข้อมูล (Backup) ก่อนนำเข้าข้อมูลจำนวนมาก
- ตรวจสอบรูปแบบไฟล์ให้ถูกต้อง
- ใช้โหมดตรวจสอบ (Dry Run) ก่อนนำเข้าจริงเสมอ
- ตรวจสอบว่ารหัสไม่ซ้ำกับข้อมูลที่มีอยู่ (ถ้าไม่ได้เลือกอัปเดต)

### ระหว่างนำเข้า

- อย่าปิดหน้าเว็บขณะนำเข้าข้อมูล
- รอให้เสร็จสิ้นก่อนทำงานอื่น
- ตรวจสอบข้อความแสดงข้อผิดพลาดอย่างละเอียด

### หลังนำเข้า

- ตรวจสอบข้อมูลที่นำเข้าในหน้าจัดการข้อมูลนั้นๆ
- ตรวจสอบประวัติการนำเข้า
- แก้ไขข้อมูลที่นำเข้าผิดพลาด (ถ้ามี)

## การจัดการข้อผิดพลาดที่พบบ่อย

### 1. รหัสซ้ำ (Duplicate Code)

**ข้อความ**: "รหัสนี้มีอยู่แล้ว" **วิธีแก้**:

- เลือก "อัปเดตข้อมูลเดิม" หากต้องการอัปเดต
- เปลี่ยนรหัสในไฟล์ หากต้องการสร้างใหม่
- เลือก "ข้ามรายการซ้ำ" หากต้องการเก็บรหัสเดิม

### 2. ฟิลด์จำเป็นไม่มีข้อมูล

**ข้อความ**: "กรุณาระบุ..." **วิธีแก้**: ตรวจสอบว่าฟิลด์ที่จำเป็น (code, name)
มีข้อมูล

### 3. รูปแบบอีเมลไม่ถูกต้อง

**ข้อความ**: "อีเมลไม่ถูกต้อง" **วิธีแก้**: ตรวจสอบรูปแบบอีเมล
(user@example.com)

### 4. ประเภทข้อมูลไม่ถูกต้อง

**ข้อความ**: "ต้องเป็น..." **วิธีแก้**: ตรวจสอบประเภทข้อมูล (type, vatType,
costingMethod)

### 5. ไฟล์ไม่สามารถอ่านได้

**ข้อความ**: "ไม่สามารถอ่านไฟล์ได้" **วิธีแก้**:

- ตรวจสอบว่าไฟล์เป็น .csv หรือ .json เท่านั้น
- สำหรับไฟล์ Excel บันทึกเป็น CSV ก่อน
- ตรวจสอบ encoding เป็น UTF-8

## API Endpoint

### POST /api/admin/import

นำเข้าข้อมูลจากไฟล์

**Request:**

- Method: POST
- Content-Type: multipart/form-data
- Body:
  - file: File (ไฟล์ CSV หรือ JSON)
  - dataType: string ('customers' | 'vendors' | 'products' | 'accounts')
  - skipDuplicates: boolean
  - updateExisting: boolean
  - dryRun: boolean

**Response:**

```json
{
  "success": true,
  "dryRun": true,
  "totalRecords": 100,
  "validCount": 95,
  "errorCount": 5,
  "preview": [
    {
      "action": "create",
      "data": { "code": "C001", "name": "..." }
    },
    {
      "action": "update",
      "data": { "code": "C002", "name": "..." }
    },
    {
      "action": "error",
      "error": "รหัสนี้มีอยู่แล้ว"
    }
  ]
}
```

### GET /api/admin/import

ดูประวัติการนำเข้าข้อมูล

**Request:**

- Method: GET
- Query Parameters:
  - dataType: string (optional) - กรองตามประเภท
  - limit: number (default: 50) - จำนวนรายการต่อหน้า
  - offset: number (default: 0) - รายการเริ่มต้น

**Response:**

```json
{
  "success": true,
  "data": [
    {
      "id": "...",
      "dataType": "customers",
      "fileName": "customers.csv",
      "fileType": "csv",
      "totalRecords": 100,
      "createdCount": 80,
      "updatedCount": 15,
      "errorCount": 5,
      "status": "COMPLETED",
      "createdAt": "2026-03-14T10:00:00Z",
      "importedBy": {
        "name": "Admin",
        "email": "admin@thaiaccounting.com"
      }
    }
  ],
  "total": 50,
  "limit": 50,
  "offset": 0
}
```

## การสำรองและกู้คืนข้อมูล

ก่อนนำเข้าข้อมูลจำนวนมาก ควรสำรองข้อมูลก่อนเสมอ:

1. ไปที่เมนู "ตั้งค่า" → "สำรองและกู้คืนข้อมูล"
2. คลิก "สำรองข้อมูล" (Backup)
3. ดาวน์โหลดไฟล์สำรองข้อมูล
4. เก็บไฟล์ไว้ในที่ปลอดภัย

หากต้องการกู้คืน:

1. ไปที่เมนู "ตั้งค่า" → "สำรองและกู้คืนข้อมูล"
2. คลิก "กู้คืนข้อมูล" (Restore)
3. เลือกไฟล์สำรองข้อมูล
4. ยืนยันการกู้คืน

## คำถามที่พบบ่อย (FAQ)

**Q: สามารถนำเข้าไฟล์ Excel (.xlsx) ได้หรือไม่?** A: ไม่รองรับโดยตรง
แต่สามารถบันทึกไฟล์ Excel เป็น CSV ก่อนนำเข้า

**Q: ขนาดไฟล์สูงสุดเท่าไหร่?** A: สูงสุด 5MB ต่อไฟล์

**Q: สามารถนำเข้ากี่รายการต่อครั้ง?** A: ไม่จำกัด แต่แนะนำไม่เกิน 1,000
รายการต่อครั้งเพื่อความเสถียร

**Q: หากนำเข้าผิดพลาดจะเกิดอะไรขึ้น?** A: ระบบจะย้อนกลับ (Rollback) ทั้งหมด
ไม่มีข้อมูลใดถูกเปลี่ยนแปลง

**Q: สามารถนำเข้าข้อมูลซ้ำได้หรือไม่?** A: ได้
ระบบจะสร้างรายการใหม่ถ้ารหัสไม่ซ้ำ หรืออัปเดตถ้าเลือก "อัปเดตข้อมูลเดิม"

**Q: ตรวจสองข้อมูลได้จากที่ไหน?** A: ไปที่เมนูตามประเภทข้อมูล (เช่น "ลูกหนี้"
สำหรับลูกค้า, "เจ้าหนี้" สำหรับผู้ขาย)

## ตัวอย่างการใช้งาน

### ตัวอย่าง 1: นำเข้าลูกค้าใหม่

```csv
code,name,taxId,address,phone,email
C001,บริษัท ก. จำกัด,1234567890123,"123 ถ.สุขุมวิท",02-123-4567,sales@companyg.com
C002,บริษัท ข. จำกัด,9876543210987,"456 ถ.พหลโยธิน",02-234-5678,sales@companyk.com
```

### ตัวอย่าง 2: นำเข้าสินค้าพร้อมสต็อก

```csv
code,name,category,unit,salePrice,costPrice,quantity,isInventory
P001,สินค้า A,หมวด A,ชิ้น,150,100,50,true
P002,สินค้า B,หมวด B,ชิ้น,200,150,30,true
```

### ตัวอย่าง 3: อัปเดตข้อมูลลูกค้าที่มีอยู่

```csv
code,name,phone,email
C001,บริษัท ก. จำกัด (อัปเดต),02-999-9999,newemail@companyg.com
```
